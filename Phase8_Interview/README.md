# Phase 8: Interview Preparation

## Learning Objectives

- Prepare for Mainframe Modernization Developer/Lead interviews
- Know core Java fundamentals with mainframe analogies
- Practice coding exercises
- Articulate your modernization experience clearly

## Interview Strategy

1. **Know the fundamentals** — Most interviews start with core Java and OOP
2. **Use mainframe analogies** — This is your differentiator; explain Java in COBOL terms
3. **Practice coding** — Reverse strings, find duplicates, use streams
4. **Tell your story** — Farmers Life migration, OneInc integration, copybook conversion

## Preparation Checklist

### Core Java (Must Know)
- [ ] String immutability, `==` vs `.equals()`
- [ ] Access modifiers (public, private, protected, default)
- [ ] static keyword (variable, method, block)
- [ ] final keyword (variable, method, class)
- [ ] Method overloading vs overriding
- [ ] Abstract class vs interface
- [ ] Exception handling (try-catch-finally)
- [ ] Autoboxing/unboxing

### Collections (Must Know)
- [ ] ArrayList vs LinkedList
- [ ] HashMap internal working
- [ ] HashMap vs TreeMap vs LinkedHashMap
- [ ] HashSet (removing duplicates)
- [ ] Collections.sort() with Comparator

### Java 8 (Must Know)
- [ ] Lambda expressions
- [ ] Stream API (filter, map, collect, reduce)
- [ ] Optional class
- [ ] Method references
- [ ] Collectors (groupingBy, toList, summingDouble)

### Spring Boot (Must Know for Modernization)
- [ ] Layered architecture (Controller -> Service -> Repository)
- [ ] Dependency Injection (@Autowired)
- [ ] REST API annotations (@GetMapping, @PostMapping, etc.)
- [ ] Entity and JPA annotations
- [ ] application.properties configuration

### Mainframe Modernization (Your Differentiator)
- [ ] Copybook to Java entity conversion
- [ ] COBOL data type to Java type mapping
- [ ] CICS to REST API translation
- [ ] Batch processing modernization approach
- [ ] Challenges and solutions from your Farmers Life experience
- [ ] How you bridged mainframe and Java teams
